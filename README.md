# Анализ удержания клиентов — Olist Brazilian E-Commerce

## 📊 Цель проекта
Выявить **сегменты клиентов по удержанию** и **источники повторных покупок** в онлайн-магазине Olist, чтобы определить, какие когорты наиболее ценны и на что должна ориентироваться маркетинг-стратегия.

## 📊 Датасет
- **Источник:** Kaggle Olist Brazilian E-Commerce
- **Период:** 2016–2018 гг.
- **Объём:** ~100k заказов, ~32k уникальных клиентов
- **Таблицы:** orders, customers, order_items, payments

## 🔍 Ключевые метрики

| Метрика | Определение |
|---------|-----------|
| **Cohort** | Когда клиент совершил первую покупку (месяц-год) |
| **Retention Rate (M+N)** | % клиентов когорты, которые вернулись спустя N месяцев |
| **Repeat Rate** | % клиентов, совершивших 2+ покупки |
| **Avg Order Value** | Средняя стоимость одного заказа по когорте |
| **Lifetime Value (LTV)** | Сумма всех покупок клиента (за доступный период) |

## 📈 Основные этапы анализа

### 1️⃣ Data Load & Cleaning
- Загрузка 4 таблиц из CSV
- Проверка пропусков и дубликатов
- Конвертация дат в `datetime`, сброс пропусков в недоставленных заказах

### 2️⃣ Cohort Analysis
- Определение первой покупки для каждого клиента
- Расчёт **когорты первой покупки** (месяца)
- Для каждого последующего месяца: % клиентов, вернувшихся
- Нормализация: retention_rate = (клиенты в М+N) / (размер когорты)

### 3️⃣ RFM & Segmentation
- Расчёт Recency, Frequency, Monetary для активных клиентов
- Выявление сегментов: VIP, Regular, At-Risk, Lost
- Анализ поведения по сегментам

### 4️⃣ Findings & Recommendations
- На что указывают низкие retention на М+3-М+6?
- Какие когорты чувствительны к сезонности?
- Что рекомендовать маркетингу?

## 📝 Примечания по качеству данных

- **2,965 пропусков в delivery_date** → заказы в статусе `shipped` или `cancelled`, исключены из анализа доставки
- **Дубликатов нет** → order_id уникален
- **Многозвёздие в join-ах** → один заказ может иметь много товаров и платежей; agregируем на уровне order_id перед когортным анализом
